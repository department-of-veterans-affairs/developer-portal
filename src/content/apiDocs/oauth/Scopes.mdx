| Scope | Values and description |
| ----- | ---------------------- |
| `profile` {.profile} | Granted by default, allows access to a user's first and last name and email. |
| `offline_access` | This scope causes the authorization server to provide a refresh token when the <HashLink to="#id-token" onClick={onHashAnchorClick}>access token</HashLink> is requested. |
| `openid` | An `id_token` is available in the authorization code flow (response_type = code) token response when the 'openid' scope is used. |
